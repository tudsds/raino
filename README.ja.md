## Raino とは

Raino は、制約付き・監査可能・ソーストレーサブルなハードウェア設計・製造引き渡しシステムです。自然言語で表現された曖昧なハードウェア要件を構造化された製品仕様に変換し、承認済みアーキテクチャを選択し、実在のサプライヤーから調達データを取得し、KiCad プロジェクトを生成し、概算コスト見積もり付きの製造バンドルを生成します。

Raino はチャットボットではありません。無制約の自律 PCB オートパイロットでもありません。すべての設計判断は承認済みアーキテクチャテンプレートに制約され、すべての部品選択はソースにトレース可能で、すべての見積もりはリアルタイムサプライヤーデータとフィクスチャ推定値を明確に区別します。システムは設計を生成する前に正式な十分性ゲートを実行し、解決できない曖昧さに直面すると停止して人間の入力を求めます。

プラットフォームは、マーケティングサイト、プロダクトスタジオアプリケーション、エンジニアリング知識用の RAG（検索拡張生成）パイプライン、見積もりエンジン、監査トレースシステム、外部サプライヤーおよび KiCad EDA ツール用のアダプターで構成されています。すべて MIT ライセンスでオープンソースです。

## なぜ Raino が必要なのか

PCB の設計は一つのステップでは完了しません。曖昧なアイデアから Gerber ファイル、部品表（BOM）、製造引き渡しに至る長いワークフローです。その過程で、部品の選択、ピン互換性の検証、エラッタの確認、フットプリントの競合解決、電気ルールチェックの実行、サプライヤーからのコスト見積もりの取得などが必要です。各ステップにはエラーの余地があり、ハードウェアのエラーは修正コストが高いです。

既存のツールの多くは、これらのステップを別々の問題として扱います。EDA ツールはレイアウトを扱いますが調達は扱いません。サプライヤー検索ツールは価格設定を扱いますが設計ルールは扱いません。チャットベースのアシスタントは部品を提案できますが、実際の回路図に基づいて検証することはできません。Raino は正式なチェックポイント、プロベナンストラッキング、フェイクデータ禁止ポリシーを備え、ワークフロー全体を一つにまとめています。部品番号を捏造せず、推定値を使用しているのにリアルタイム価格だと主張せず、重要な曖昧さを飛ばして尋ねずに進むことはありません。

## 外部 API と境界

Raino は複数の外部システムと対話します。各システムは定義済み契約を持つ厳格な境界として扱われます。

### KiCad（GPL 境界）

KiCad は外部の GPL ライセンス EDA ツールです。Raino は KiCad コードを組み込みません。`packages/kicad-worker-client` で定義された CLI コマンド（`kicad-cli`）を通じて通信します。生成された KiCad デザインや KiCad ライブラリファイルは、Raino 自体とは異なるライセンス条項を持つ場合があります。

### サプライヤーアダプター

Raino は `packages/supplier-clients` で DigiKey、Mouser、JLCPCB 用のアダプターインターフェースを定義しています。インターフェース + アダプターパターンを使用します。リアルタイム API 認証情報は想定しません。認証情報がない場合、システムはフィクスチャモードで実行され、推定値が明確にラベル付けされます。

### RAG 境界

`packages/rag` の RAG システムは、エンジニアリングドキュメントのエンベディングと検索を処理します。プラガブルインターフェースを通じてエンベディングサービスに接続します。フィクスチャモードでは、インメモリストレージを使用します。

## RAG のスコープと非スコープ

### RAG が対象とするもの

- データシートとコンポーネント仕様
- メーカーが発行するエラッタドキュメント
- アプリケーションノートとリファレンスデザイン
- パッケージアウトラインとランドパターン
- 設計判断のためのエンジニアリング知識検索

### RAG が対象としないもの

- リアルタイム価格データ
- 在庫状況
- 最小注文数量
- 注文処理
- ドキュメントインジェストの追従速度を超える変化の速いデータ

リアルタイム価格、在庫、注文はサプライヤーアダプターが排他的に処理します。見積もりの信頼できる情報源は常に構造化されたサプライヤーアダプター出力であり、RAG 検索結果ではありません。

## 十分性ゲート

Raino は設計を生成する前に正式な十分性チェックを実行します。このゲートは、必要なデータがすべて存在し矛盾がないことを検証します。チェック内容：

- 各候補部品の公式データシートが存在する
- ベンダーがエラッタを発行している場合、エラッタが存在する
- 複雑な部品にアプリケーションノートが存在する
- パッケージとランドパターンのドキュメントが利用可能
- 構造化調達フィールドが入力されている
- 各 BOM 行にフットプリントマッピングが存在する
- カスタム以外のすべての部品に代替品が存在する
- ドキュメント間に未解決の矛盾がない
- 重要なプレースホルダーが残っていない

十分性ゲートが失敗した場合、Raino は具体的なギャップを報告し、設計生成に進みません。ユーザーはギャップを解決するか、リスクを明示的に受け入れてから続行する必要があります。

## KiCad Worker 境界

Raino は KiCad を組み込み依存ではなく外部ワーカーとして使用します。境界の動作：

1. `packages/kicad-worker-client` が CLI コマンド契約とジョブタイプを定義
2. `services/design-worker` がジョブを KiCad CLI に送信
3. KiCad が `.kicad_pro`、`.kicad_sch`、`.kicad_pcb`、エクスポートファイルを生成
4. Raino が ERC（電気ルールチェック）と DRC（設計ルールチェック）で出力を検証
5. KiCad 出力からプレビューアセット（SVG、PDF、GLB）を生成

Raino リポジトリには KiCad ソースコードは一切含まれません。KiCad ライブラリと生成されたデザインには、Raino の MIT ライセンスとは別のライセンス条項が適用される場合があります。

## プレビューとダウンロードフロー

設計が検証を通過した後、Raino は複数のプレビューとダウンロードアーティファクトを生成します：

| アーティファクト  | 形式      | 説明                                        |
| ----------------- | --------- | ------------------------------------------- |
| 回路図プレビュー  | SVG、PDF  | 回路図レンダリング                          |
| PCB 2D プレビュー | SVG       | 表面と裏面の銅箔レイヤービュー              |
| PCB 3D プレビュー | GLB       | インタラクティブ 3D ボードモデル            |
| BOM エクスポート  | CSV、JSON | 代替品付き完全な部品表                      |
| ガーバーファイル  | RS-274X   | 製造対応 PCB ファイル                       |
| 製造バンドル      | ZIP       | ガーバー、BOM、実装ファイル、ドリルファイル |

すべてのアーティファクトはマニフェストにチェックサムとプロベナンスメタデータを持ちます。

## 見積もりフロー

`services/quote-worker` の見積もりエンジンは、3つの信頼度バンドを持つ概算コスト見積もりを生成します。

### 計算式

```
小計 = 設計自動化費
     + エンジニアリングレビュー費
     + PCB 製造見積もり
     + 部品見積もり
     + 組立見積もり
     + QA 包装梱包費

中間見積もり = 小計 + 予備費 + 利益
下限見積もり = 中間見積もり * 0.8
上限見積もり = 中間見積もり * 1.25
```

### 費用構成

| 項目                       | デフォルト値              |
| -------------------------- | ------------------------- |
| 設計自動化費               | $500                      |
| エンジニアリングレビュー費 | $300                      |
| PCB 製造                   | $0.10/cm²（推定基板面積） |
| 組立                       | $0.05/部品実装            |
| QA、包装、梱包             | $100 ベース + $0.50/個    |
| 予備費                     | 小計の 10%                |
| 利益                       | （小計 + 予備費）の 15%   |

### 信頼度レベル

| レベル | 基準                                            |
| ------ | ----------------------------------------------- |
| **高** | すべての BOM 行にリアルなサプライヤー価格がある |
| **中** | 70%以上の行にリアル価格、30%以下が推定値        |
| **低** | 30%を超える行が推定価格を使用                   |

すべての見積もりには仮定の完全なリストが含まれます。部品価格がリアルタイムサプライヤークエリではなくフィクスチャデータから来ている場合、見積もりは推定として明確にフラグ付けされます。

## Raino 引き渡しフロー

概算見積もりが生成された後、ユーザーはオプションで Raino に PCBA（プリント基板実装）見積もりを依頼できます。この引き渡しフロー：

1. ユーザーが概算見積もりと仮定を確認
2. ユーザーが「Raino に PCBA 見積もりを依頼」をクリック
3. Raino が見積もり ID と希望数量で注文インテントを送信
4. ステータストラッキング付きの引き渡しレコードが作成
5. ユーザーが確認を受け取り、進捗を追跡可能

これはオプションのステップです。ユーザーはすべての製造アーティファクトをダウンロードし、任意の基板製造業者に依頼できます。

## はじめ方

### 前提条件

- Node.js >= 20.0.0
- pnpm >= 9.0.0
- Git

### クローンとインストール

```bash
git clone https://github.com/tudsds/raino.git
cd raino
pnpm install
pnpm dev
```

### 起動内容

`pnpm dev` 実行後、2つのアプリケーションが起動します：

- **マーケティングサイト**：`http://localhost:3000`
- **プロダクトスタジオ**：`http://localhost:3001`

初期セットアップにサプライヤー API キーは不要です。キーがない場合、システムはラベル付き推定値でフィクスチャモードで実行されます。

## テストと監査

### テストコマンド

```bash
# すべてのテストを実行
pnpm test

# ユニットテストのみ
pnpm test:unit

# 統合テストのみ
pnpm test:integration

# E2E テストのみ
pnpm test:e2e
```

### テスト要件

Raino は厳格なテスト基準を適用します：

- すべての Zod スキーマはパース成功とパース失敗の両方のテストを持つこと
- 見積もりエンジンは正確な数値結果を検証するゴールデンアウトプットテストを持つこと
- インジェストパイプラインはパス/フェイル動作を確認する十分性ゲートテストを持つこと
- RAG 検索はソース帰属を確認するプロベナンス検証テストを持つこと
- 捏造部品、欠落エラッタ、矛盾するドキュメントなどの失敗モードをテストすること

### 監査トレース

監査ワーカーはすべての重要なアクションを記録します。トレース内容：

- ソースプロベナンス付きの BOM 決定
- 部品選択の根拠
- チェックサム付きアーティファクトマニフェスト
- 見積もりの仮定と信頼度レベル
- ポリシー違反フラグ

監査トレースは `GET /api/projects/:id/audit` の API から検査可能です。

## デプロイメント

Raino は Turborepo ビルドオーケストレーションによる Vercel デプロイメント向けに設計されています。

### 手順

1. リポジトリを GitHub にフォークまたはプッシュ
2. GitHub リポジトリを Vercel に接続
3. ルートディレクトリを `/`（モノレポルート）に設定
4. Vercel が `vercel.json` から Next.js アプリを自動検出
5. Vercel ダッシュボードで環境変数を設定
6. デプロイ

### プレビューと本番

- すべてのプルリクエストが自動プレビューデプロイメントを取得
- main ブランチへのマージが本番デプロイメントをトリガー

### ビルド設定

ルートの `vercel.json` は以下を指定：

- `buildCommand`：`pnpm build`
- `installCommand`：`pnpm install`
- Turborepo が内部ビルド順序を処理

## コントリビューション

コントリビューションを歓迎します。手順：

1. リポジトリをフォーク
2. `main` からフィーチャーブランチを作成
3. テスト付きで変更を加える
4. `pnpm typecheck`、`pnpm lint`、`pnpm test` を実行して検証
5. 明確な説明付きでプルリクエストを作成

### コードスタンダード

- TypeScript 厳格モード。`as any`、`@ts-ignore`、`@ts-expect-error` は禁止
- 空 catch ブロックは禁止
- すべてのパブリック関数はテストを持つこと
- 既存のプロジェクト構造とパターンに従うこと
- コミットメッセージは変更内容を明確に記述すること

### 避けるべきアンチパターン

- ビルドを通すためにテストを削除する
- ユーザー要件をデモに簡略化する
- 対応するテストなしに機能を追加する
- 認証情報やシークレットをコミットする

## よくある質問

**Raino はチャットボットですか？**

いいえ。Raino は自然言語インテイクステップを持つ構造化ワークフローシステムです。正式な状態マシン、十分性ゲート、監査トレースを持つ制約付きエージェントを使用します。オープンエンドのレスポンスを生成したり、未検証の設計判断をしたりしません。

**Raino を使うのに KiCad のインストールが必要ですか？**

設計生成を伴わないローカル開発では不要です。実際の KiCad プロジェクトを生成するには、KiCad がインストールされ、`kicad-cli` が PATH で利用可能である必要があります。KiCad がない場合、デザインワーカーはモックモードで実行されます。

**見積もりはどの程度正確ですか？**

見積もりは概算です。正確さは信頼度レベルに依存します。「高」信頼度はほとんどの価格がリアルタイムサプライヤーデータから来ていることを意味します。「低」信頼度はほとんどの価格がフィクスチャ推定値であることを意味します。すべての見積もりには詳細な仮定リストが含まれており、自分で判断できます。

**自分のサプライヤーを使って、Raino の引き渡しサービスを使わないことはできますか？**

はい。すべての製造アーティファクト（ガーバー、BOM、実装ファイル）はダウンロード可能です。任意の PCB 製造業者や実装業者に送付できます。

**サプライヤー API が利用できない場合、どうなりますか？**

Raino はフィクスチャモードにフォールバックし、影響を受けるデータをすべて推定値として明確にラベル付けします。サイレントに劣化することはありません。監査トレースはどのデータがフィクスチャから来たか、いつ来たかを正確に記録します。

**Raino はアナログ設計もサポートしていますか、それともデジタルだけですか？**

Raino はアーキテクチャテンプレートシステムに収まる設計をサポートします。現在のブートストラップは一般的なデジタルボードアーキテクチャに焦点を当てていますが、テンプレートシステムはアナログおよびミックスドシグナル設計に拡張可能です。

**Raino と ChatGPT で PCB を設計するのとどう違いますか？**

ChatGPT は部品を提案したり概念を説明したりできますが、ERC/DRC の実行、KiCad プロジェクトの生成、リアルタイムサプライヤー価格のクエリ、トレース可能な監査トレースの維持はできません。Raino は LLM 推論を構造化されたエンジニアリングツールと正式なチェックポイントと組み合わせます。

**Raino は完全にオフラインで実行できますか？**

はい。フィクスチャデータとモックエンベディングで、システム全体がオフラインで動作します。リアルタイム価格やリアルタイムサプライヤーデータはありませんが、設計ワークフロー、検証、アーティファクト生成はすべてネットワークアクセスなしで機能します。

## プロベナンスとセキュリティ

### フェイク統合禁止ポリシー

Raino は絶対に：

- リアルタイム価格や在庫データを捏造しない
- フィクスチャデータ使用時にリアルタイム API 接続があると主張しない
- 報告なしにサイレントに劣化モードにフォールバックしない
- ユーザー確認なしに重要な曖昧さを飛ばして進まない

すべての劣化モードパスは監査トレースから検査可能です。

### プロベナンストラッキング

Raino のすべての設計判断はトレース可能：

- 部品選択はデータシート、エラッタ、アプリケーションノートにリンク
- BOM エントリはサプライヤー、価格ソース、価格が推定かどうかを記録
- アーティファクトはチェックサムと生成タイムスタンプ付きのマニフェストを持つ
- 見積もりは仮定の完全なリストと信頼度スコアを含む
- ポリシー違反は表面化され記録される

### 入力バリデーション

すべての API 入力は Zod スキーマで検証されます。型安全なパースと明確なエラーメッセージ。型付けされていないデータはシステムに入りません。

### 外部境界

- KiCad は定義済み CLI 契約を通じてのみ通信
- サプライヤーアダプターはインターフェース + アダプターパターンを使用し、直接データベース接続なし
- エンベディングサービスはプラガブルインターフェースを通じて接続し、モックフォールバック付き
