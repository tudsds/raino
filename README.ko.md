## Raino란 무엇인가?

Raino는 제약 기반, 감사 가능, 소스 추적 가능한 하드웨어 설계 및 제조 인계 시스템입니다. 자연어로 표현된 모호한 하드웨어 요구사항을 구조화된 제품 사양으로 변환하고, 승인된 아키텍처를 선택하고, 실제 공급업체에서 조달 데이터를 확인하며, KiCad 프로젝트를 생성하고, 대략적인 비용 견적과 함께 제조 번들을 생성합니다.

Raino는 챗봇이 아닙니다. 제약 없는 자율 PCB 오토파일럿도 아닙니다. 모든 설계 결정은 승인된 아키텍처 템플릿으로 제한되고, 모든 부품 선택은 소스까지 추적 가능하며, 모든 견적은 실시간 공급업체 데이터와 픽스처 추정치를 명확하게 구분합니다. 시스템은 설계를 생성하기 전에 정식 충분성 게이트를 실행하고, 해결할 수 없는 모호성에 직면하면 멈춰서 사람의 입력을 요청합니다.

플랫폼은 마케팅 사이트, 제품 스튜디오 애플리케이션, 엔지니어링 지식용 RAG(검색 증강 생성) 파이프라인, 견적 엔진, 감사 추적 시스템, 외부 공급업체 및 KiCad EDA 도구용 어댑터로 구성됩니다. 모든 코드는 MIT 라이선스로 오픈소스입니다.

## 왜 Raino가 필요한가?

PCB 설계는 한 단계로 끝나지 않습니다. 모호한 아이디어에서 게르버 파일, BOM(부품 명세서), 제조 인계에 이르는 긴 워크플로우입니다. 그 과정에서 부품 선택, 핀 호환성 확인, 에라타 검토, 풋프린트 충돌 해결, 전기 규칙 검사 실행, 공급업체 비용 견적 확보 등이 필요합니다. 각 단계마다 오류의 여지가 있고, 하드웨어 오류의 수정 비용은 높습니다.

기존 도구들은 대부분 이 단계들을 별개의 문제로 취급합니다. EDA 도구는 레이아웃을 다루지만 조달은 다루지 않습니다. 공급업체 검색 도구는 가격을 다루지만 설계 규칙은 다루지 않습니다. 챗 기반 어시스턴트는 부품을 추천할 수 있지만 실제 회로도 기반으로 검증할 수는 없습니다. Raino는 공식 체크포인트, 출처 추적, 가짜 데이터 금지 정책을 갖추고 전체 워크플로우를 하나로 통합합니다. 부품 번호를 조작하지 않고, 추정치를 사용하면서 실시간 가격이라고 주장하지 않으며, 중요한 모호성을 묻지 않고 건너뛰지 않습니다.

## 외부 API 및 경계

Raino는 여러 외부 시스템과 상호작용합니다. 각 시스템은 정의된 계약이 있는 엄격한 경계로 처리됩니다.

### KiCad (GPL 경계)

KiCad는 외부 GPL 라이선스 EDA 도구입니다. Raino는 KiCad 코드를 임베드하지 않습니다. `packages/kicad-worker-client`에 정의된 CLI 명령(`kicad-cli`)을 통해 통신합니다. 생성된 KiCad 설계와 KiCad 라이브러리 파일은 Raino 자체와 다른 라이선스 조항이 적용될 수 있습니다.

### 공급업체 어댑터

Raino는 `packages/supplier-clients`에서 DigiKey, Mouser, JLCPCB용 어댑터 인터페이스를 정의합니다. 인터페이스 + 어댑터 패턴을 사용합니다. 실시간 API 자격 증명은 가정하지 않습니다. 자격 증명이 없으면 시스템은 픽스처 모드로 실행되며 추정치가 명확하게 라벨링됩니다.

### RAG 경계

`packages/rag`의 RAG 시스템은 엔지니어링 문서의 임베딩과 검색을 처리합니다. 플러그형 인터페이스를 통해 임베딩 서비스에 연결합니다. 픽스처 모드에서는 인메모리 스토리지를 사용합니다.

## RAG의 범위와 비범위

### RAG가 다루는 것

- 데이터시트 및 컴포넌트 사양
- 제조사가 발행하는 에라타 문서
- 애플리케이션 노트 및 레퍼런스 디자인
- 패키지 아웃라인 및 랜드 패턴
- 설계 결정을 위한 엔지니어링 지식 검색

### RAG가 다루지 않는 것

- 실시간 가격 데이터
- 재고 가용성
- 최소 주문 수량
- 주문 처리
- 문서 인제스트가 따라갈 수 없는 속도로 변하는 데이터

실시간 가격, 재고, 주문은 공급업체 어댑터가 전담하여 처리합니다. 견적의 신뢰할 수 있는 출처는 항상 구조화된 공급업체 어댑터 출력이지 RAG 검색 결과가 아닙니다.

## 충분성 게이트

Raino는 설계를 생성하기 전에 정식 충분성 검사를 실행합니다. 이 게이트는 필요한 모든 데이터가 존재하고 일관성이 있는지 검증합니다. 검사 내용:

- 각 후보 부품의 공식 데이터시트가 존재하는지
- 벤더가 에라타를 발행하는 경우 에라타가 존재하는지
- 복잡한 부품에 애플리케이션 노트가 존재하는지
- 패키지 및 랜드 패턴 문서가 사용 가능한지
- 구조화된 조달 필드가 입력되었는지
- 각 BOM 행에 풋프린트 매핑이 존재하는지
- 커스텀이 아닌 모든 부품에 대체품이 존재하는지
- 문서 간에 해결되지 않은 모순이 없는지
- 중요한 플레이스홀더가 남아 있지 않은지

충분성 게이트가 실패하면 Raino는 구체적인 누락 항목을 보고하고 설계 생성으로 진행하지 않습니다. 사용자는 누락 항목을 해결하거나 위험을 명시적으로 수락해야 계속 진행할 수 있습니다.

## KiCad Worker 경계

Raino는 KiCad를 임베드된 의존성이 아닌 외부 워커로 사용합니다. 경계 동작 방식:

1. `packages/kicad-worker-client`가 CLI 명령 계약과 작업 유형을 정의
2. `services/design-worker`가 작업을 KiCad CLI로 전송
3. KiCad가 `.kicad_pro`, `.kicad_sch`, `.kicad_pcb` 및 익스포트 파일을 생성
4. Raino가 ERC(전기 규칙 검사)와 DRC(설계 규칙 검사)로 출력을 검증
5. KiCad 출력에서 프리뷰 에셋(SVG, PDF, GLB)을 생성

Raino 저장소에는 KiCad 소스 코드가 전혀 포함되지 않습니다. KiCad 라이브러리와 생성된 설계는 Raino의 MIT 라이선스와 별개의 라이선스 조항이 적용될 수 있습니다.

## 프리뷰 및 다운로드 플로우

설계가 검증을 통과한 후, Raino는 여러 프리뷰 및 다운로드 아티팩트를 생성합니다:

| 아티팩트      | 형식      | 설명                              |
| ------------- | --------- | --------------------------------- |
| 회로도 프리뷰 | SVG, PDF  | 회로도 렌더링                     |
| PCB 2D 프리뷰 | SVG       | 상하면 동박 레이어 뷰             |
| PCB 3D 프리뷰 | GLB       | 인터랙티브 3D 보드 모델           |
| BOM 익스포트  | CSV, JSON | 대체품이 포함된 전체 부품 명세서  |
| 게르버 파일   | RS-274X   | 제조 준비된 PCB 파일              |
| 제조 번들     | ZIP       | 게르버, BOM, 실장 파일, 드릴 파일 |

모든 아티팩트는 체크섬과 출처 메타데이터가 포함된 매니페스트를 갖습니다.

## 견적 플로우

`services/quote-worker`의 견적 엔진은 3개의 신뢰도 밴드가 있는 대략적 비용 견적을 생성합니다.

### 계산식

```
소계 = 설계 자동화 수수료
     + 엔지니어링 검토 수수료
     + PCB 제조 추정치
     + 부품 추정치
     + 조립 추정치
     + QA 포장 처리비

중간 견적 = 소계 + 예비비 + 이윤
하한 견적 = 중간 견적 * 0.8
상한 견적 = 중간 견적 * 1.25
```

### 비용 구성

| 항목                   | 기본값                     |
| ---------------------- | -------------------------- |
| 설계 자동화 수수료     | $500                       |
| 엔지니어링 검토 수수료 | $300                       |
| PCB 제조               | $0.10/cm² (추정 기판 면적) |
| 조립                   | $0.05/부품 실장            |
| QA, 포장, 처리         | $100 기본 + $0.50/개       |
| 예비비                 | 소계의 10%                 |
| 이윤                   | (소계 + 예비비)의 15%      |

### 신뢰도 수준

| 수준     | 기준                                         |
| -------- | -------------------------------------------- |
| **높음** | 모든 BOM 행에 실제 공급업체 가격이 있음      |
| **중간** | 70% 이상의 행에 실제 가격, 30% 이하가 추정치 |
| **낮음** | 30%를 초과하는 행이 추정 가격을 사용         |

모든 견적에는 가정의 전체 목록이 포함됩니다. 부품 가격이 실시간 공급업체 쿼리가 아닌 픽스처 데이터에서 온 경우, 견적은 추정치로 명확하게 표시됩니다.

## Raino 인계 플로우

대략적 견적이 생성된 후, 사용자는 선택적으로 Raino에 PCBA(인쇄회로기판 조립) 견적을 요청할 수 있습니다. 이 인계 플로우:

1. 사용자가 대략적 견적과 가정을 검토
2. 사용자가 "Raino에 PCBA 견적 요청"을 클릭
3. Raino가 견적 ID와 원하는 수량으로 주문 의도를 제출
4. 상태 추적이 포함된 인계 레코드가 생성
5. 사용자가 확인을 받고 진행 상황을 추적 가능

이것은 선택적 단계입니다. 사용자는 모든 제조 아티팩트를 다운로드하고 원하는 제조사에 의뢰할 수 있습니다.

### 테스트 요구사항

Raino는 엄격한 테스트 표준을 적용합니다:

- 모든 Zod 스키마는 파싱 성공과 파싱 실패 테스트를 모두 가져야 합니다
- 견적 엔진은 정확한 수치 결과를 검증하는 골든 아웃풋 테스트를 가져야 합니다
- 인제스트 파이프라인은 통과/실패 동작을 확인하는 충분성 게이트 테스트를 가져야 합니다
- RAG 검색은 소스 귀속을 확인하는 출처 검증 테스트를 가져야 합니다
- 조작된 부품, 누락된 에라타, 모순되는 문서 등의 실패 모드를 테스트해야 합니다

감사 워커는 모든 중요한 작업을 기록합니다. 추적 내용:

- 소스 출처가 포함된 BOM 결정
- 부품 선택 근거
- 체크섬이 포함된 아티팩트 매니페스트
- 견적 가정 및 신뢰도 수준
- 정책 위반 플래그

감사 추적은 `GET /api/projects/:id/audit` API에서 검사할 수 있습니다.

## 배포

Raino는 Turborepo 빌드 오케스트레이션을 통한 Vercel 배포용으로 설계되었습니다.

### 단계

1. 저장소를 GitHub에 포크 또는 푸시
2. GitHub 저장소를 Vercel에 연결
3. 루트 디렉토리를 `/`(모노레포 루트)로 설정
4. Vercel이 `vercel.json`에서 Next.js 앱을 자동 감지
5. Vercel 대시보드에서 환경 변수 설정
6. 배포

### 프리뷰 및 프로덕션

- 모든 풀 리퀘스트가 자동 프리뷰 배포를 받습니다
- main 브랜치로의 머지가 프로덕션 배포를 트리거합니다

### 빌드 설정

루트의 `vercel.json`은 다음을 지정합니다:

- `buildCommand`: `pnpm build`
- `installCommand`: `pnpm install`
- Turborepo가 내부 빌드 순서를 처리합니다

## 자주 묻는 질문

**Raino는 챗봇인가요?**

아닙니다. Raino는 자연어 인테이크 단계가 있는 구조화된 워크플로우 시스템입니다. 정식 상태 머신, 충분성 게이트, 감사 추적을 갖춘 제약 기반 에이전트를 사용합니다. 개방형 응답을 생성하거나 검증되지 않은 설계 결정을 내리지 않습니다.

**Raino를 사용하려면 KiCad를 설치해야 하나요?**

설계 생성이 없는 로컬 개발에는 필요하지 않습니다. 실제 KiCad 프로젝트를 생성하려면 KiCad가 설치되어 있고 `kicad-cli`가 PATH에서 사용 가능해야 합니다. KiCad가 없으면 디자인 워커가 모크 모드로 실행됩니다.

**견적은 얼마나 정확한가요?**

견적은 대략적인 추정치입니다. 정확도는 신뢰도 수준에 따라 다릅니다. "높음" 신뢰도는 대부분의 가격이 실시간 공급업체 데이터에서 왔음을 의미합니다. "낮음" 신뢰도는 대부분의 가격이 픽스처 추정치임을 의미합니다. 모든 견적에는 직접 판단할 수 있는 상세한 가정 목록이 포함됩니다.

**Raino의 인계 서비스 대신 자체 공급업체를 사용할 수 있나요?**

네. 모든 제조 아티팩트(게르버, BOM, 실장 파일)를 다운로드할 수 있습니다. 원하는 PCB 제조사나 조립 업체에 보낼 수 있습니다.

**공급업체 API를 사용할 수 없으면 어떻게 되나요?**

Raino는 픽스처 모드로 폴백하고 영향을 받는 모든 데이터를 추정치로 명확하게 라벨링합니다. 조용히 저하되지 않습니다. 감사 추적은 어떤 데이터가 픽스처에서 왔는지, 언제 왔는지 정확하게 기록합니다.

**Raino는 아날로그 설계도 지원하나요, 디지털만인가요?**

Raino는 아키텍처 템플릿 시스템에 맞는 설계를 지원합니다. 현재 부트스트랩은 일반적인 디지털 보드 아키텍처에 초점을 맞추고 있지만, 템플릿 시스템은 아날로그 및 혼합 신호 설계로 확장 가능합니다.

**Raino와 ChatGPT로 PCB를 설계하는 것과 어떤 차이가 있나요?**

ChatGPT는 부품을 추천하고 개념을 설명할 수 있지만, ERC/DRC 실행, KiCad 프로젝트 생성, 실시간 공급업체 가격 조회, 추적 가능한 감사 추적 유지는 할 수 없습니다. Raino는 LLM 추론을 구조화된 엔지니어링 도구 및 공식 체크포인트와 결합합니다.

**Raino를 완전히 오프라인에서 실행할 수 있나요?**

네. 픽스처 데이터와 모크 임베딩으로 전체 시스템이 오프라인에서 작동합니다. 실시간 가격이나 실시간 공급업체 데이터는 없지만, 설계 워크플로우, 검증, 아티팩트 생성은 모두 네트워크 액세스 없이 기능합니다.

## 출처 추적 및 보안

### 가짜 통합 금지 정책

Raino는 절대:

- 실시간 가격이나 재고 데이터를 조작하지 않습니다
- 픽스처 데이터를 사용할 때 실시간 API 연결이 있다고 주장하지 않습니다
- 보고 없이 조용히 저하 모드로 폴백하지 않습니다
- 사용자 확인 없이 중요한 모호성을 건너뛰지 않습니다

모든 저하 모드 경로는 감사 추적에서 검사 가능합니다.

### 출처 추적

Raino의 모든 설계 결정은 추적 가능합니다:

- 부품 선택은 데이터시트, 에라타, 애플리케이션 노트에 링크
- BOM 항목은 공급업체, 가격 소스, 가격이 추정치인지 여부를 기록
- 아티팩트는 체크섬과 생성 타임스탬프가 포함된 매니페스트를 가짐
- 견적은 가정의 전체 목록과 신뢰도 점수를 포함
- 정책 위반은 표시되고 기록됨

### 입력 검증

모든 API 입력은 Zod 스키마로 검증됩니다. 타입 안전한 파싱과 명확한 에러 메시지. 타입이 지정되지 않은 데이터는 시스템에 들어가지 않습니다.

### 외부 경계

- KiCad는 정의된 CLI 계약을 통해서만 통신
- 공급업체 어댑터는 인터페이스 + 어댑터 패턴을 사용하며, 직접 데이터베이스 연결 없음
- 임베딩 서비스는 플러그형 인터페이스를 통해 연결하며, 모크 폴백 포함
